Es una arquitectura de software donde todos los componentes y funcionalidades están integrados en un solo programa o aplicación. Suelen ser más difíciles de escalar y mantener en comparación con arquitecturas modulares o microservicios. Ejemplo: Aplicaciones empresariales tradicionales como ERP antiguos.

Es un sistema de archivos utilizado en sistemas operativos Windows Soporta características avanzadas como permisos de seguridad, cifrado, recuperación ante fallos y compresión. Es más robusto que FAT32 y exFAT en términos de gestión de datos y seguridad.

Es un sistema informático que proporciona servicios o recursosa otros dispositivos (clientes) en una red. Pueden ser físicos o virtuales y pueden ejecutar múltiples funciones, como alojamiento web, bases de datos, almacenamiento, etc. Ejemplos: Servidores web (Apache, Nginx), servidores de correo (Exchange, Postfix).

Es un modelo de seguridad informática donde el acceso a la información se divide en distintos niveles según la sensibilidad de los datos. Se usa en sistemas militares, bancarios y empresariales para restringir el acceso a información sensible. Ejemplo: Un sistema de seguridad con niveles "Confidencial", "Secreto" y "Público".

Conjunto de computadoras interconectadas que trabajan como un solo sistema.Mejora la escalabilidad, tolerancia a fallos y rendimiento. Ejemplo: Redes P2P, bases de datos distribuidas como Cassandra o Google File System.

Modelo de entrega de servicios de computación (almacenamiento, procesamiento, software) a través de internet.Puede ser pública, privada o híbrida. Ejemplo: AWS, Google Cloud, Microsoft Azure.
